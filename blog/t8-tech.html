Regular Expression

By Wei Wen September 18, 2015.

In theoretical computer science and formal language theory, a regular expression (abbreviated regex or regexp and sometimes called a rational expression) is a sequence of characters that define a search pattern, mainly for use in pattern matching with strings, or string matching, i.e. "find and replace"-like operations. Many programming languages provide regular expression capabilities, some built-in, for example Perl, JavaScript, Ruby, AWK, and Tcl, and others via a standard library, for example .NET languages, Java, Python and C++ (since C++11). Most other languages offer regular expressions via a library. The list below contains the most common regular expression sympols:

  • . - Matches any character, except for line breaks if dotall is false.
  • * - Matches 0 or more of the preceding character.
  • + - Matches 1 or more of the preceding character.
  • \d - Matches any single digit
  • ? - Preceding character is optional. Matches 0 or 1 occurrence.
  • \w - Matches any word character (alphanumeric & underscore).
  • [XYZ] - Matches any single character from the character class.
  • [XYZ]+ - Matches one or more of any of the characters in the set.
  • $ - Matches the end of the string.
  • ^ - Matches the beginning of a string.
  • [^a-z] - When inside of a character class, the ^ means NOT; in this case, match anything that is NOT a lowercase letter.

I think the best way to learn how to use regular expression is to read as many examples as possible. Therefore, next I will show couple of examples and explain what they means. 


            var username = 'JohnSmith';
            /[A-Za-z_-]+/.test(username); // returns true

Here, the regualar expression is checking for letters uppercase [A-Z], lowercase [a-z], an underscore [_], and a dash [-]. Since variable "username" contains the right expression, it passes the test and return ture. 


            var str = 'this is my string';
            str.split(/\s/); // return ["this", "is", "my", "string"]

You're most likely already familiar with the split method. It accepts a single regular expression which represents where the "split" should occur. Please note that we can also use a string if we'd prefer. In the example above, by passing "\s" - representing a single space - we've now split our string into an array. Of course, If you need to access one particular value, just append the desired index - str.split(/\s/)[3]. 


            var username = 'J;ohnSmith;@%';
            username = username.replace(/[^A-Za-z\d_-]+/, '');
            username; // return JohnSmith;@%

            var username = 'J;ohnSmith;@%';
            username = username.replace(/[^A-Za-z\d_-]+/g, '');
            username; // return JohnSmith

This is another useful application of regular application. In the first code example, it is saying replacing any expressions in variable "username" are not letter, digit, underscore, and dash with ''. However, as the result shown, only the first unmatch expression is replaced. In order to replace all of them, you can add "g" at the end which means global search. By doing so, we can the result we actually want.

There are so much more other usages of regular expression. Mastering on it will make your programming life much easier.If you want to see more example, you can take a look at Jeffrey Way's You Don't Know Anything About Regular Expressions: A Complete Guide.
